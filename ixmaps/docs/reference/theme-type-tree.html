Theme type tree — all in one

The whole szFlag hierarchy as a single nested tree — type dispatch, CHART's sub-shape dispatch, and SYMBOL's multi-value alignments down to PLOT, indented top to bottom. Vertical scroll only.

solid — a real branch: the engine tests the string and picks exactly one path
dashed — orthogonal: reads from the same string, but never chooses geometry
  • type:the szFlag string
    • FEATUREraw geographic items, no classing computed — draws PointLineStringMultiLineStringPolygonMultiPolygonSphere(not MultiPoint / GeometryCollection)
    • CHOROPLETHpolygons filled by class — needs a FEATURE layer for the geometry to color; combines directly as type:FEATURE|CHOROPLETH — see color selection modes ↓
    • CHARTa drawn shape per feature — positioned via lookupfield:, either by coordinates straight from the data (lat|lon fields or a GeoJSON point) or joined to a FEATURE layer's own shape by id
      • BAR / BARSone card per value on a shared axis
      • PIE+ DONUT / STARBURST — radial, segment per value
      • BUBBLE+ SQUARE / LABEL — one value, one sized symbol
      • SYMBOLsingle value = one named shape; 2,060 lines, the big one
        • multi-value → | SEQUENCE, one symbol per value
        • CENTERsuperposed — shared center, each sized by its value (default)
        • BASEvertical pile, each symbol stacked on the previous
        • HORZside-by-side row
        • TOP / BOTTOMaligned to the top / bottom edge
        • CONEdiagonal pile, climbing upward
        • STARfirst value at the center, the rest orbit it as satellites
        • RANDOMscattered around the point at even angles
        • RINGSvalues accumulate into concentric rings, biggest first
        • PLOTvalues become positions — the sequence turns into a chart
          • PLOTX / PLOTYvalues run along the x / the y axis
          • PLOTXY / PLOTYXscatter from explicit xfield / yfield columns
          • PLOTVARdeviation from the mean, like DEVIATION
          • LINESconnects the points with line segments instead of discrete symbols
            • AREAfills a closed area between the line and the baseline
              • FADEfades that fill toward the baseline instead of a solid color
            • line style: STACKEDZEROISVALUEZEROISNOTVALUENOCLIPINLINETEXT
          • LOLLIPOPadds a stem from the baseline up to each point — works with or without LINES
          • last-point markers: LASTPOPLASTARROWLASTVALUE
          • curve & scale shaping: LOGINVERTSMOOTHGRADIENTLINREG
          • GRID + BOXboth together → a titled, framed sparkline
      • VECTOR / BEZIERa connector between two node positions — needs a second position, often looked up on a FEATURE layer
      • Waffle, Buffer & the minimal primitivessmaller, more specialized branches of the same per-item drawing loop
        • WAFFLEone unit square per count — a countable alternative to BAR
        • BUFFERnot a value chart — a fixed real-world dimension; doubles as a selection tool
        • DOT / QUADthe "most simple representation" — no modifiers apply
        • BLANKdraws nothing — a test hook
        • USERyou draw the SVG, the engine supplies the data
          • userdraw:myChartnames your function — resolves to ixmaps.myChart
          • called as fn(SVGDocument, opt) once per feature → returns {x, y} to offset the result
          • opt gives you: targetthemeitemdbRecordvaluevaluessizemaxSizecolortextcolorccolorclassflagmark
          • myChart_init / myChart_finishoptional once-before / once-after hooks — slimmer opt: {target, theme}
      • BOXthe one orthogonal that's visible: background, border, title
        • cross-shape modifiers — layered on whichever shape was drawn: Aggregation modeValue pipelineSize exponentRendering contextAnimationStrokeMulti-chart layoutDraw order
    • WMSan external map image overlay — fetches from an ArcGIS-style export endpoint (server:), sized and positioned live from the current map view, not from a style option
    • color selection modes— colors CHOROPLETH / CHART's fill, chooses no geometry
      • CLASSESquantity → N ordered classes
        • EQUIDISTANTequal-width classes (the default)
        • QUANTILEequal-count classes
        • NATURALbreaks that follow the data's own gaps
        • HEADTAILrecursive mean-split — the odd one out, ignores classes:N entirely
        • LOGequal-width on a log-transformed scale
        • POW2equal-width on a squared scale
        • POW3equal-width on a cubed scale
      • CATEGORICALone hue per distinct value
      • DOMINANTone field wins, lends its color
      • COMPOSECOLORseveral fields blended at once
    • value combination— how records landing on the same position get combined
      • AGGREGATEgates the whole combine-multiple-records routine — without it, none of the below runs
        • spatial binning (optional) — snaps positions to a grid before matching, cell width in real-world meters or constant screen pixels (recalculated every zoom): gridwidthgridwidthpx
        • SUMadds the values (the default)
        • MEANalso divides by the item count
        • MAXkeeps a running maximum instead of summing
        • MINkeeps a running minimum instead of summing — symmetric with MAX
      • GROUPthe same spatial binning as AGGREGATE, but without combining values — items stay separate, just shared and ordered within their bin
      • RELOCATErecomputes the merged position as the average of the original points — values untouched
    • legend display— the separate legend-panel renderer (ui/js/tools/legend.js), not the per-item drawing engine above
      • NOLEGENDdrops the theme from the legend panel entirely
      • SIMPLELEGENDone compact inline line (swatch + label) instead of the default two-line layout
      • COMPACTLEGENDcurrently identical to SIMPLELEGEND — no distinguishing behavior in the code
      • TEXTLEGENDkeeps the label, drops the color swatch
